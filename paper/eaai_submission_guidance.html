P0 Critical

Desk-Rejection Blockers

Fix these before deeper rewriting. EAAI states that manuscripts missing these conditions can be desk-rejected before peer review.

  • Rewrite the abstract to 250 words or fewer. State purpose, principal results, and major conclusions in a concise factual style.
  • Make the abstract explicitly name the AI contribution and the engineering application. AI contribution: physics-informed hybrid model and TabPFN-based formulation factor. Engineering application: cigarette combustion, aerosol filtration, and mainstream-smoke constituent prediction.
  • Remove or define acronyms in the title and abstract. The current title begins with CIGAR before definition; EAAI forbids undefined acronyms in title and abstract.
  • Confirm single-column formatting, page count, and file size. Keep the manuscript in single-column format, final PDF at or below 50 pages, and submission package under 100 MB.
  • Prepare a defensible public-data or restricted-data position. If raw tobacco data cannot be public, explain confidentiality constraints and release processed features, code, split files, and result summaries where possible.
Current evidence: main_NNK.tex uses a 277-word abstract, unnumbered iScience-style sections, numbered citations, and an "available upon request" data/code statement.
P1 High

Journal Repositioning

Shift the paper from a specialized life-science framing to a practical engineering-AI contribution.

  • Reduce the broad public-health opening. Keep enough motivation, but lead earlier with the engineering problem: interpretable prediction for a combustion and filtration system.
  • Strengthen engineering-AI related work. Add or foreground hybrid physics-informed ML, interpretable ML, surrogate modeling, tabular foundation models, and AI for engineered process systems.
  • Tighten the novelty claim. Emphasize native puff-resolved and segment-resolved physical trajectories, not only scalar yield accuracy.
  • Moderate harm-reduction language. Prefer "regulatory assessment," "emissions prediction," and "design-to-emission simulation" unless health outcomes are directly validated.
P2 Medium

Manuscript Structure

Convert the article from iScience conventions to EAAI's numbered manuscript format.

Recommended section order

  • 1. Introduction
  • 2. Materials and methods
  • 3. CIGAR model
  • 4. Experimental setup
  • 5. Results
  • 6. Discussion
  • 7. Conclusions

Current source to convert

The manuscript currently uses Abstract, Introduction, Results, Discussion, Resource availability, Acknowledgments, Declaration of interests, and STAR Methods.

  • Replace all unnumbered main headings with numbered sections. Do not number the abstract or back-matter declarations.
  • Move STAR Methods content into Methods, Model, and Experimental Setup. Keep enough detail for independent reproduction.
  • Add a standalone Conclusions section. Separate final conclusions from limitations and future work.
  • Move long tables or auxiliary details to supplementary material if page count is tight. Prioritize model comparison, mechanism example, and key dataset details in the main article.
P3 Medium

Submission Files

EAAI uses double-anonymized review and asks for several separate editable files.

  • Create a separate title page. Include article title, author names, affiliations, full postal address for the corresponding author, email, acknowledgments, funding, and declarations.
  • Create a separate anonymized manuscript. Remove author names, affiliations, acknowledgments, institutional identifiers where possible, and any self-identifying supplementary material.
  • Add 1 to 6 keywords. Candidate keywords: physics-informed learning; interpretable AI; tabular regression; combustion modeling; smoke emissions; hybrid modeling.
  • Create a separate highlights file. Use 3 to 5 bullets, each no more than 85 characters including spaces.
  • Prepare a graphical abstract if the team wants one. Adapt the existing flowchart into a concise readable image. Preferred size is at least 531 x 1328 pixels or proportional.
P4 High

Declarations

Declarations need special care because the study involves tobacco institutions and tobacco-industry funding.

  • Reassess the competing-interest statement. Do not rely on a generic "no competing interests" statement if employment, affiliations, funding, or sponsor relationships require disclosure.
  • Add a complete funding statement. State China National Tobacco Corporation grant number and the sponsor's role in design, data collection, analysis, writing, and submission.
  • Add CRediT author contributions. Use roles such as Conceptualization, Data curation, Methodology, Software, Validation, Visualization, Supervision, and Writing.
  • Add a generative AI declaration if AI tools were used. Use Elsevier's format: tool name, purpose, author review/editing, and author responsibility.
  • Strengthen the data and code availability statement. Prefer a repository link or DOI. If not possible, explain the restriction clearly and identify what can be shared.
P5 Supporting

Formatting And References

These items are less strategic than the fit and disclosure issues, but they should be clean before submission.

  • Consider switching from numbered citations to author-year citations. The current source uses numbered natbib with unsrtnat; EAAI examples use author-year style.
  • Verify complete reference metadata and DOI availability. Check author names, titles, journal names, years, pagination, DOIs, and access dates for web references.
  • Number displayed equations if cross-referenced. Use consecutive equation numbering in the main text and appendix-specific numbering if appendices are added.
  • Submit figures as separate logical files. Use names such as Figure_1.pdf and Figure_2.pdf. Ensure captions explain all symbols and abbreviations.
  • Remove Windows metadata sidecar files before packaging. Do not submit files ending in Zone.Identifier.
P6 Supporting

Cover Letter

The cover letter should explain why the manuscript now belongs in an engineering-AI journal.

  • Lead with the real-world engineering application. Frame the problem as process-level AI prediction for engineered combustion and aerosol filtration.
  • Name the AI novelty in one sentence. Example: CIGAR combines a differentiable mechanistic release model with a tabular foundation model to produce interpretable per-puff trajectories.
  • State originality and submission status. Confirm the work is not under consideration elsewhere and all authors approved the submission.
  • Do not mention the iScience rejection. Use that history only internally to guide repositioning.
